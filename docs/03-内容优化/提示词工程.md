# 提示词工程

GEO 时代，内容团队有一个新的杠杆：**用提示词提升内容质量与结构化程度**。

这一章不是讲"怎么用 AI 写内容"，而是讲：
- 怎么用提示词帮自己把内容想得更清楚
- 怎么用提示词把内容结构做得更利于 AI 理解
- 怎么把提示词沉淀成团队的内容生产资产

---

## 一、为什么内容团队需要懂提示词

很多人对提示词的理解还停留在"让 AI 帮我写文章"。

但在 GEO 语境下，提示词的价值远不止于此：

### 1. 提示词是"思考脚手架"

好的提示词能帮你：
- 把模糊的想法变成清晰的结构
- 把零散的观点组织成完整的逻辑链
- 把隐性的判断变成显性的表达

这个过程本身，就是在提升内容质量。

### 2. 提示词是"结构化助手"

GEO 要求内容结构清晰、段落单任务、结论前置。

提示词可以帮你：
- 检查内容结构是否完整
- 识别哪些段落信息过载
- 建议更清晰的表达顺序

### 3. 提示词是"质量检查员"

在内容发布前，用提示词做一次"GEO 友好度检查"：
- 标题是否对应真实问题
- 开头是否前置结论
- 关键概念是否有明确定义
- 是否有足够的 E-E-A-T 信号

### 4. 提示词是"团队资产"

当团队把有效的提示词沉淀下来：
- 新人可以更快上手内容标准
- 不同人写的内容风格更一致
- 内容质量波动更小

## 二、GEO 内容生产的提示词框架

我们把 GEO 内容生产拆成五个环节，每个环节都有对应的提示词策略：

```
选题 → 大纲 → 写作 → 优化 → 检查
```

### 环节一：选题与问题定义

**目标**：确保内容对应真实用户问题，而不是自说自话。

**提示词示例**：

```
你是一个 GEO 内容策略顾问。我要写一篇关于 [主题] 的内容。

请帮我：
1. 列出用户在这个主题下最常问的 5-8 个问题
2. 按问题类型分类（定义型、方法型、评估型、对比型、趋势型）
3. 指出哪些问题最适合用长文回答，哪些问题适合用短文回答
4. 建议一个最能对应核心问题的标题

主题：[填写你的主题]
目标读者：[填写你的读者类型]
内容形式：[长文/短文/问答/案例]
```

**使用建议**：
- 不要直接采用 AI 生成的标题，而是用它来检验你的选题是否对应真实问题
- 如果 AI 列出的问题和你设想的不一样，说明你可能需要调整选题角度

---

### 环节二：大纲与结构设计

**目标**：确保内容结构完整、逻辑清晰、便于 AI 抽取信息单元。

**提示词示例**：

```
你是一个内容结构顾问。我要写一篇关于 [主题] 的长文，核心问题是 [核心问题]。

请帮我设计一个 GEO 友好的内容大纲，要求：
1. 开头第一段要能直接回答核心问题（结论前置）
2. 每个小节只讲一件事（段落单任务）
3. 包含定义、原因、方法、案例、注意事项等完整信息单元
4. 小标题要语义清晰，能独立传达该节内容
5. 最后要有核心结论总结

请按以下格式输出：
- 标题建议（3 个选项）
- 开头结论段（100-150 字）
- 大纲结构（一级标题 + 二级标题 + 每节核心要点）
- 建议补充的 E-E-A-T 信号（案例、数据、经验等）
```

**使用建议**：
- 用这个提示词生成的不是"最终大纲"，而是"结构检查参考"
- 对比 AI 建议和你的原始想法，找出结构上的盲点

---

### 环节三：写作与表达优化

**目标**：让表达更清晰、更结构化、更利于 AI 理解。

**提示词示例（段落优化）**：

```
你是一个内容编辑。请帮我优化下面这段内容，让它更清晰、更结构化：

[粘贴你的段落]

优化要求：
1. 如果一段话讲了多件事，请拆分成多个段落
2. 把核心结论放在段落开头
3. 把模糊表达改成明确判断
4. 如果有列表信息，请用列表格式呈现
5. 保持原意，不要添加你没有把握的信息

输出格式：
- 优化后的内容
- 修改说明（你改了哪里，为什么这样改）
```

**提示词示例（概念定义）**：

```
请帮我把下面这个概念用更清晰的方式定义：

[粘贴你的概念描述]

要求：
1. 用 1-2 句话给出核心定义
2. 说明这个概念不是什么（划清边界）
3. 给出一个具体例子
4. 说明这个概念的适用场景和局限

输出要能让 AI 直接引用作为"标准定义"。
```

**使用建议**：
- 不要直接用 AI 重写的内容，而是用它来发现表达上的问题
- 重点关注"哪些地方 AI 认为不够清晰"，这往往是需要改进的地方

---

### 环节四：结构化增强

**目标**：让内容更容易被 AI 抽取信息单元。

**提示词示例**：

```
你是一个信息架构师。请分析下面这篇内容，指出如何让它更结构化：

[粘贴你的内容]

请从以下维度分析：
1. 信息单元识别：这篇文章包含哪些信息单元（定义、结论、步骤、比较、案例、数据、观点等）
2. 结构清晰度：哪些段落信息过载，需要拆分
3. 可引用性：哪些句子可以直接被 AI 引用作为答案
4. 改进建议：具体到段落级别的修改建议

请用表格形式输出分析结果。
```

**使用建议**：
- 这个提示词适合在内容初稿完成后使用
- 重点关注"哪些句子可以直接被 AI 引用"，这是 GEO 内容的核心标准

---

### 环节五：GEO 友好度检查

**目标**：发布前做一次系统性检查。

**提示词示例**：

```
你是一个 GEO 内容审核员。请对下面这篇内容进行 GEO 友好度评估：

[粘贴你的内容]

请按以下检查清单逐项评估：

【标题检查】
- 标题是否对应真实用户问题
- 标题是否清楚传达内容主题
- 标题是否避免过度营销化表达

【结构检查】
- 开头是否在前 100 字给出核心结论
- 每个段落是否只讲一件事
- 小标题是否语义清晰
- 是否有完整的信息单元（定义、原因、方法、案例、结论）

【可引用性检查】
- 是否有可以直接被 AI 引用的定义句
- 是否有可以直接被 AI 引用的结论句
- 是否有可以直接被 AI 引用的步骤清单
- 是否有可以直接被 AI 引用的案例摘要

【E-E-A-T 检查】
- 是否有经验信号（个人/团队实践经验）
- 是否有专业信号（行业认知、方法论）
- 是否有可信信号（数据、案例、第三方背书）
- 是否有权威信号（引用来源、专家观点）

【改进建议】
- 列出 3-5 个最优先的改进点
- 每个改进点给出具体修改建议

请用评分 + 文字说明的方式输出（每项 1-5 分）。
```

**使用建议**：
- 这个提示词适合在内容发布前使用
- 不要追求每项都 5 分，而是找出最弱的 2-3 项优先改进

## 三、提示词模板沉淀

当团队找到有效的提示词后，应该把它沉淀成可复用的资产。

### 提示词模板结构

一个好的提示词模板应该包含：

```markdown
# [模板名称]

## 使用场景
[什么时候用这个模板]

## 角色设定
[AI 扮演什么角色]

## 输入变量
- [变量 1]：[说明]
- [变量 2]：[说明]

## 提示词正文
[完整的提示词]

## 输出格式
[希望 AI 按什么格式输出]

## 使用注意事项
[使用时的注意事项和常见误区]

## 迭代记录
- v1.0 [日期]：初始版本
- v1.1 [日期]：优化了 XX 部分
```

### 建议沉淀的模板类型

内容团队建议至少沉淀以下几类模板：

| 模板类型 | 使用场景 | 使用频率 |
|----------|----------|----------|
| 选题分析 | 确定内容方向时 | 每篇内容前 |
| 大纲设计 | 写长文前 | 每篇长文前 |
| 段落优化 | 内容粗糙时 | 按需 |
| 概念定义 | 需要明确定义时 | 按需 |
| GEO 检查 | 发布前检查 | 每篇内容前 |
| 标题优化 | 标题不确定时 | 按需 |
| 案例结构化 | 整理案例时 | 按需 |

## 四、提示词工作流整合

提示词不应该是一个"额外步骤"，而应该嵌入到现有工作流中。

### 个人创作者工作流

```
选题 → [提示词：选题分析] → 大纲 → [提示词：大纲设计] → 
写作 → [提示词：段落优化] → 检查 → [提示词：GEO 检查] → 发布
```

### 小团队工作流

```
选题会 → 确定主题 → [提示词：选题分析] → 
分工写作 → [提示词：大纲设计] → 
互审 → [提示词：GEO 检查] → 
修改 → 发布
```

### 大团队工作流

```
内容规划 → 主题池 → [提示词模板库支持] → 
内容生产 → [标准化提示词检查] → 
质量审核 → [GEO 检查清单] → 
发布 → 效果追踪
```

**关键原则**：
- 提示词是辅助，不是替代
- 提示词应该嵌入现有流程，而不是增加额外步骤
- 提示词模板应该持续迭代，不是一成不变

## 五、提示词使用的常见误区

### 误区一：直接用 AI 生成的内容发布

**问题**：AI 生成的内容往往缺少真实经验和判断。

**建议**：
- 用提示词帮自己思考，而不是替代思考
- AI 生成的内容要经过人工审核和修改
- 保持内容的"人味"和经验感

### 误区二：提示词越复杂越好

**问题**：过长的提示词往往效果不稳定。

**建议**：
- 一个提示词聚焦一个任务
- 复杂任务拆成多个提示词顺序执行
- 定期精简和优化提示词

### 误区三：一套提示词用到底

**问题**：不同内容类型需要不同的提示词策略。

**建议**：
- 为不同类型内容设计不同提示词
- 根据反馈持续迭代提示词
- 建立团队的提示词模板库

### 误区四：忽视提示词的"思考价值"

**问题**：只把提示词当成"生成工具"，忽略了它作为"思考脚手架"的价值。

**建议**：
- 关注 AI 的分析和建议，不只是生成结果
- 用 AI 的反馈来改进自己的思考方式
- 把提示词当成"内容顾问"，不只是"写作工具"

## 六、对中国团队的特别建议

### 1. 用中文思维写提示词

很多提示词模板是英文的，直接翻译过来效果不好。

建议：
- 用中文直接写提示词
- 考虑中文表达习惯和阅读偏好
- 针对中文内容的特点设计检查项

### 2. 重视业务价值表达

中国读者更关注"能不能马上用"。

在提示词中增加：
- 业务场景说明
- 执行步骤细化
- 预期效果描述

### 3. 考虑组织语境

中国团队的内容生产往往是协作的。

提示词设计要考虑：
- 多人协作的内容一致性
- 不同角色的内容审核标准
- 内容资产的沉淀和复用

## 七、提示词效果评估

怎么知道你的提示词好不好用？

### 评估维度

| 维度 | 评估问题 | 评估方法 |
|------|----------|----------|
| 思考辅助 | 提示词是否帮你把问题想得更清楚 | 主观感受 + 内容质量对比 |
| 结构改善 | 提示词是否让内容结构更清晰 | GEO 检查评分对比 |
| 效率提升 | 提示词是否节省了时间 | 生产周期对比 |
| 质量稳定 | 提示词是否让内容质量更稳定 | 不同人生产的内容对比 |
| 可复用性 | 提示词是否能被团队其他人使用 | 团队反馈 |

### 迭代建议

- 每使用 10 次，回顾一次提示词效果
- 收集团队成员的反馈和建议
- 定期更新提示词模板库

## 八、本章核心结论

1. 提示词在 GEO 时代是内容团队的新杠杆，不只是"让 AI 写内容"
2. 提示词的核心价值是"思考脚手架"和"结构化助手"
3. 建议把 GEO 内容生产拆成五个环节，每个环节都有对应的提示词策略
4. 有效的提示词应该沉淀成团队模板，持续迭代
5. 提示词应该嵌入现有工作流，而不是增加额外步骤
6. 提示词是辅助，不是替代——保持内容的"人味"和经验感

## 实践建议

如果你现在想开始用提示词提升内容质量，建议从这三个模板开始：

1. **GEO 检查提示词**：发布前检查每篇内容
2. **大纲设计提示词**：写长文前先设计结构
3. **段落优化提示词**：优化表达不清的段落

用熟这三个，再逐步扩展到其他环节。

提示词的价值不在于"用得多"，而在于"用得对"。
