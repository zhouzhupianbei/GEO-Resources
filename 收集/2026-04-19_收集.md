## 📊 搜索引擎优化：从 SEO 到 GEO 的范式转移

### 核心指标对比

| 指标类型 | SEO 时代 | GEO 时代 | 变化幅度 |
|---------|---------|---------|---------|
| **核心目标** | 搜索排名 | AI 引用 | 范式转移 |
| **关键指标** |  organic 流量 | 露出率 (ER) | 新指标 |
| **成功定义** | 第一页排名 | 首推回答 | 质变 |
| **内容策略** | 关键词优化 | 语义完整性 | 深度升级 |
| **技术重点** | 页面速度 | RAG 可解析性 | 新维度 |
| **评估周期** | 周/月 | 实时/日 | 10x 加速 |

### GEO 效果评估指标体系

```
一级指标 (战略层)
├── 露出率 (ER) = 品牌露出次数 / 相关查询总量
├── 首推率 (FR) = 首位推荐次数 / 露出总次数
└── 正面率 (SR) = 正面露出次数 / 露出总次数

二级指标 (战术层)
├── 引用深度 = 平均引用内容长度 / 内容总长度
├── 引用准确率 = 准确引用次数 / 总引用次数
├── 知识图谱覆盖率 = 实体被收录数 / 目标实体总数
└── 语义关联度 = 品牌与核心概念共现频率

三级指标 (执行层)
├── Schema 标记覆盖率
├── 内容更新频率
├── 权威外链数量
└── 多模态内容占比
```

## ⚠️ 风险预警：GEO 优化常见陷阱

### 陷阱 1: 过度优化触发 AI 降权

**风险表现**:
- 短时间内大量修改核心页面
- 关键词/语义密度异常集中
- Schema 标记与内容不匹配

**规避策略**:
- 采用渐进式优化 (每周<20% 页面)
- 保持自然语言风格
- 确保标记与内容一致

**恢复周期**: 2-4 周

### 陷阱 2: 忽视负面信息管理

**风险表现**:
- 只关注正面内容发布
- 未监测 AI 回答中的负面引用
- 缺乏危机响应机制

**规避策略**:
- 建立负面监测告警 (日度)
- 准备危机公关预案
- 主动发布澄清/更正内容

**恢复周期**: 4-12 周

### 陷阱 3: 技术债务积累

**风险表现**:
- Schema 标记过期未更新
- llms.txt 文件无法访问
- 站点速度持续下降

**规避策略**:
- 建立技术健康度周报
- 设置自动化监测告警
- 季度技术债务清理

**恢复周期**: 1-3 周
